#기술 스택

Python Streamlit JSON Selenium RegEx

프로젝트 소개

이 프로젝트는 수원시의 전통 노포 식당 정보를 JSON 데이터셋으로 검색하는 웹 기반 애플리케이션입니다. 기획부터 개발, 배포까지 전 과정을 1인 개발로 진행했으며, 백엔드(데이터 처리 및 검색 로직)와 프론트엔드(사용자 인터페이스)를 모두 직접 담당했습니다.

목적

사용자에게 식당명, 주소, 메뉴, 가격 검색 기능을 제공하고, Streamlit 기반의 사용자 친화적인 인터페이스를 제공하는 것이 주된 목적입니다. 특히, 메뉴 검색 시 가격 필터링 및 URL 디코딩된 주소 출력을 지원합니다. 이 프로젝트는 모든 기능을 1인 개발로 구현함으로써 프로젝트 전반에 대한 이해도와 문제 해결 능력을 입증하는 것을 목표로 합니다.

역할 및 성과

데이터 크롤링부터 전처리, 적재, 그리고 대시보드 개발까지 모든 과정을 1인 책임지고 수행.

  • 데이터 수집 및 전처리: 다이닝코드(DiningCode) 사이트에서 Selenium을 활용하여 노포 식당 데이터를 직접 수집. 수집된 데이터를 JSON 파일로 저장하기 위해 필요한 인코딩/디코딩 과정을 직접 처리하며 데이터 정제 및 가공.
  • 대시보드 성능 최적화: Streamlit 대시보드의 캐싱 기능을 적극 활용하여 데이터 로딩 및 처리 속도를 획기적으로 개선.
  • 통합적인 UI/UX 디자인 및 구현: Streamlit 대시보드의 전체적인 UI/UX 디자인 직접 담당 및 구현.
  • 데이터-서비스 간 원활한 연동: GitHub에 호스팅된 JSON 파일과 Streamlit 기반 프론트엔드 간의 완벽한 데이터 연결 및 흐름 구축.

레슨 런 (Lessons Learned)

  • 데이터 처리와 기본기 중요성: JSON 데이터 파싱 및 처리 능력 향상. 가격 데이터의 예외 상황(누락/비정상) 정규 표현식 처리 기술 습득. 이 모든 과정에서 견고한 기본기 중요성 깊이 체감.
  • Streamlit 웹 개발 및 UI/UX 경험: Streamlit 활용 동적 웹 애플리케이션 구축. 컬럼 및 사이드바 레이아웃, 사용자 입력 처리 기술 학습. 직접 UI/UX 설계 및 구현, 사용자 관점 개발 중요성 깊이 이해.
  • 문제 해결 및 독립적 개발 능력: 프로젝트 모든 단계 문제 스스로 분석 및 해결. 독립적 문제 해결 능력과 전체 개발 프로세스 총괄 역량 크게 향상.

#기술 스택

Python Pandas Numpy Scikit-learn FastText eKoNLPy Kiwipiepy Matplotlib Seaborn NLP Text Mining Naive Bayes N-gram TF-IDF CountVectorizer

프로젝트 소개

방대한 양의 경제 뉴스 기사, 한국은행 금융통화위원회(MPB) 의사록, 금융 보고서 등 비정형 텍스트 데이터를 텍스트 마이닝 기법으로 분석하여 금리 변화 방향을 예측하는 모델을 개발한 프로젝트입니다.

목적

중앙은행 커뮤니케이션의 매파적/비둘기파적 '톤'을 정량화하고, 이를 예측 모델에 활용하여 금리 인상 및 하락 예측의 정확도를 향상시키는 것이 목표입니다. 특히, 텍스트 기반 지표가 기존 거시경제 변수만으로는 포착하기 어려웠던 정책 결정의 추가적인 설명력을 제공함을 보였습니다.

프로젝트 수행 과정

  • 대규모 경제 텍스트 데이터 수집: 연합인포맥스, 이데일리, 한국경제 뉴스 및 한국은행 금통위 의사록 등 총 20만 건 이상의 금리 관련 텍스트 데이터 수집.
  • 고도화된 한국어 텍스트 전처리: 품사 필터링, 불용어 제거, 띄어쓰기 교정 등 한국어 특성에 최적화된 전처리 파이프라인 구축.
  • 매파/비둘기파 감성 사전 구축 및 분류: eKoNLPy, Kiwipiepy, FastText를 활용해 텍스트 토큰화 및 단어 임베딩 수행. 매파적(Hawkish) 성격과 비둘기파적(Dovish) 성격을 나누는 감성 사전 구축. 코사인 유사도와 같은 기법을 사용해 개별 단어가 두 사전 중 어느 쪽에 더 가까운지 분류하고, 이를 통해 의사록(정답셋)의 감성 톤 예측.
  • 금리 예측 모델링: N-gram 특징과 Naive Bayes Classifier(NBC)를 중심으로 CountVectorizer, TF-IDF 기법을 활용한 금리 변화 예측 모델 개발.
  • 탐색적 데이터 분석(EDA) 및 시각화: 총 수집 기사 수, 데이터 처리 파이프라인, FastText 단어 임베딩 산점도, 각 모델 학습 결과 및 성능 비교 등 시각적 분석 자료 제공.

역할

  • 데이터 수집 및 전처리 리더: 약 3만 건에 달하는 연합인포맥스 뉴스 기사 크롤링 및 전처리 과정 총괄. 품사 필터링, 불용어 제거, 띄어쓰기 교정 등 고품질 데이터 정제 작업 수행하여 모델 학습 기반 마련했습니다.
  • 텍스트 임베딩 및 모델 학습 전문가: FastText 모델 학습 주도하여 서브워드 학습, OOV(Out-Of-Vocabulary) 처리, Rare Word(저빈도 단어) 대응 등 임베딩 성능 최적화. 이를 통해 텍스트 데이터가 모델에 효과적으로 활용될 수 있도록 기여했습니다.

성과

  • 고성능 금리 예측 지표 개발: 매파/비둘기파 감성 사전 구축 및 코사인 유사도 기반 분류를 통해 한국은행의 통화정책 '톤' 성공적 정량화. 이 텍스트 기반 지표는 기존 거시경제 변수 대비 한국은행의 통화정책 결정에 대한 높은 설명력을 가짐을 입증.
  • 강화된 예측 모델 성능: 텍스트 마이닝을 통해 얻은 정량적 정보가 금리 인상/하락 예측 모델의 설명력과 예측 정확도를 유의미하게 개선.
  • 텍스트 데이터 활용 능력 증명: 대규모 비정형 텍스트 데이터의 수집, 전처리, 임베딩, 모델링 전 과정에 대한 숙련된 기술 활용 및 복잡한 문제 해결 역량 성공적으로 증명.

레슨 런 (Lessons Learned)

  • 한국어 자연어 처리의 중요성: 한국어 텍스트의 복잡한 특성을 고려한 eKoNLPy와 같은 경제·금융 특화 NLP 도구의 활용이 분석 정확도를 극대화하는 데 필수적임을 체감.
  • 분야별 감성 사전 구축의 필요성: 범용 사전으로는 포착하기 어려운 경제·금융 분야의 미묘한 의미 차이를 정확히 반영하기 위해 특정 분야에 특화된 감성 사전 구축 및 유사도 기반 분류의 중요성을 깊이 학습.
  • 모델 선택 및 최적화 전략: Word2Vec, FastText 등 다양한 단어 임베딩 기법과 N-gram, NBC 등 모델링 기법의 특성 이해 및 실제 데이터에 최적화된 조합을 찾아 예측 성능을 지속적으로 개선하는 경험을 얻음.

#기술 스택

Python Streamlit MySQL MariaDB Pandas OpenAI GPT NLTK Plotly SQL

프로젝트 소개

이 프로젝트는 스팀(Steam)에서 게임 정보와 리뷰를 수집하여 DB에 저장하고, OpenAI GPT를 활용한 감성 분석, 사용자 태그 기반 추천 게임, 그리고 Streamlit 대시보드로 시각화한 팀 프로젝트입니다.

목적

스팀 게임 데이터와 리뷰를 분석하여 사용자에게 유용한 정보와 맞춤형 게임 추천을 제공하는 것이 주된 목적입니다. 특히, 다양한 태그를 기반으로 게임을 필터링하고, 리뷰 감성 분석을 통해 게임의 긍정적/부정적 요소를 명확히 제시하여 사용자 경험을 향상하는 데 중점을 둡니다.

역할

  • 대시보드 개발 및 DB 연동: 팀원들이 수집하고 분석한 데이터를 MySQL/MariaDB에서 가져와 Streamlit 기반의 대시보드에 연결하고 시각화 기능을 구현했습니다.
  • UI/UX 설계 및 구현: 아이디어 제공 담당자(팀장)와의 와이어프레임 협의를 통해 사용자 요구사항을 반영하고, 이를 바탕으로 Streamlit 대시보드의 전체적인 UI/UX 디자인을 직접 담당 및 구현했습니다.
  • 성능 최적화: 대시보드 성능 향상을 위해 캐싱(Caching) 기능을 적극 활용하여 데이터 로딩 및 처리 속도를 개선했습니다.

성과

  • 대시보드 성능 최적화: Streamlit 대시보드의 캐싱 기능을 적극 활용하여 데이터 로딩 및 처리 속도를 획기적으로 개선. 불필요한 DB 접근을 최소화하고 대시보드의 반응성을 대폭 향상.
  • 통합적인 UI/UX 디자인 및 구현: 아이디어 제공 담당자(팀장)와의 와이어프레임 협의를 통해 사용자 요구사항을 반영하고, 이를 바탕으로 Streamlit 대시보드의 전체적인 UI/UX 디자인 직접 담당 및 구현. 직관적인 레이아웃과 데이터 시각화 방식을 통해 사용자 친화적인 인터페이스 제공.
  • 데이터-서비스 간 원활한 연동: 팀원들이 구축한 MySQL/MariaDB 데이터베이스와 Streamlit 기반 프론트엔드 간의 완벽한 데이터 연결 및 흐름 구축. 사용자의 인터랙션에 따라 DB에서 필요한 데이터를 효율적으로 조회하고, 이를 대시보드에 실시간으로 반영하여 안정적인 서비스 운영 환경 구현.
  • 고도화된 리뷰 키워드 분석 시각화: OpenAI GPT를 통해 분석된 리뷰 데이터를 기반으로 긍정/부정 키워드 워드클라우드, 바 차트, 버블 차트 등 다양한 시각화 요소를 구현하여 복잡한 리뷰 감성 분석 결과 한눈에 파악.

레슨 런 (Lessons Learned)

  • Streamlit 웹 개발 및 UI/UX 경험: 태그 필터링, 키워드 분석, 추천 게임 조회, 리뷰 상세 보기 등 대시보드 기능 전반 개발 및 구현. Streamlit의 강력한 시각화 및 인터랙티브 기능 활용 능력 습득. 직접 UI/UX 설계 및 구현, 사용자 관점 개발 중요성 깊이 이해.
  • DB 연동 및 데이터 처리: 팀원들이 구축한 MySQL/MariaDB 데이터베이스와의 연동을 통해 대용량 데이터 효율적 조회 및 처리 경험. 데이터베이스 스키마 이해 및 쿼리 최적화 실무 적용.
  • 협업을 통한 프로젝트 완성: 각자 역할 분담을 통한 데이터 수집, 분석, 대시보드 통합 등 팀 협업의 중요성 경험. 타 팀원 결과물 활용을 통한 최종 산출물 완성 과정에서 협업 역량 강화.

#기술 스택

Python Streamlit Pandas Numpy Scikit-learn XGBoost LSTM ARIMA SMOTE Folium GeoJSON Plotly EDA

프로젝트 소개

과거 범죄 데이터, 기상 데이터, 경제 지표, 검색량 데이터를 통합하여 이상동기 범죄(묻지마 범죄) 발생 가능성을 예측하고, Streamlit 기반 인터랙티브 대시보드를 통해 지역별 실시간 경보 시스템을 제공하는 프로젝트입니다.

목적

이상동기 범죄의 발생 가능성을 사전에 파악하여 지역 주민과 당국에 실시간 경보 및 지역별 위험 정보를 제공함으로써 범죄 예방과 공공 안전을 강화하는 것을 목표로 합니다. 특히, 기상, 경제, 검색량 데이터를 결합해 범죄 발생의 환경적·사회적 요인을 정량화하고 예측 정확도를 높였습니다.

프로젝트 수행 과정

  • 다양한 데이터 수집 및 통합: 2015~2025년 범죄 데이터(시군구 단위), 기상청 데이터(습도, 기온 등), 네이버 데이터랩 검색량, 경제 지표(실업률 등)를 수집. 약 10년간의 데이터를 통합해 분석 기반 구축.
  • 데이터 전처리 및 매핑: 지역 데이터를 광역시도 단위로 재분류하고, 기상 및 범죄 데이터를 250×250m 격자 단위로 정규화. 결측값 처리 및 한국어 특성에 맞춘 검색량 데이터 전처리 수행.
  • 예측 모델 개발: RandomForest, XGBoost with SMOTE, LSTM, ARIMA를 활용해 범죄 발생 가능성을 예측. 특히, SMOTE로 데이터 불균형 문제를 해결하고, LSTM으로 검색량 시계열 데이터를 분석해 시간적 패턴 도출.
  • Streamlit 대시보드 구현: Folium과 GeoJSON을 활용해 지역별 위험도를 인터랙티브 지도에 시각화. 4단계 위험 경보 체계(Safe, Warning, Danger, Very Dangerous)를 임계값(0.6, 0.7, 0.85)에 따라 설정해 사용자 친화적 대시보드 제공.
  • 탐색적 데이터 분석(EDA) 및 시각화: 기상 데이터와 범죄 발생 간 상관관계, 검색량과 범죄 건수의 시차 분석(±3일, ±7일)을 통해 패턴 분석. Plotly를 활용한 시계열 그래프 및 지역별 히트맵 생성.

역할 (팀 리더)

  • 데이터 전처리 및 통합: 2015~2025년 범죄, 기상, 검색량 데이터를 수집하고, 광역시도 및 격자 단위로 매핑. 결측값 처리와 데이터 정규화를 주도하여 모델 학습 기반 마련했습니다.
  • 모델링 및 최적화 전문가: XGBoost와 SMOTE를 활용한 이진 분류 모델 개발, LSTM으로 시계열 데이터 분석. 하이퍼파라미터 튜닝과 임계값 최적화를 통해 예측 성능 개선했습니다.
  • 대시보드 개발자: Streamlit과 Folium을 사용해 지역별 위험 경보를 시각화. GeoJSON 기반 자치구 경계 지도 구현 및 4단계 경보 체계 설계했습니다.

성과

  • 정확한 범죄 예측 모델 개발: 기상, 경제, 검색량 데이터를 통합한 XGBoost와 LSTM 모델로 이상동기 범죄 발생 가능성을 성공적으로 예측. 특히, SMOTE 적용으로 소수 클래스 예측 성능 개선.
  • 사용자 친화적 대시보드 구현: Streamlit 기반 대시보드를 통해 지역별 위험도를 직관적으로 시각화. 4단계 경보 체계로 실시간 정보 제공, 공공 안전 강화에 기여.
  • 데이터 기반 인사이트 도출: 기상 데이터(습도, 기온)와 검색량 데이터의 시차 분석을 통해 범죄 발생의 환경적·사회적 요인 규명. 상관계수(0.54~0.58) 기반 패턴 분석으로 예측 설명력 향상.

레슨 런 (Lessons Learned)

  • 다양한 데이터 통합의 중요성: 범죄, 기상, 경제, 검색량 등 이종 데이터를 통합하고 정규화하는 과정에서 데이터 단위와 시간적 일관성 유지가 예측 성능에 큰 영향을 미침을 학습.
  • 모델 최적화의 필요성: SMOTE, 하이퍼파라미터 튜닝, 임계값 조정을 통해 데이터 불균형과 과적합 문제를 해결하며 모델 성능을 최적화하는 경험을 얻음.
  • 시각화의 효과: Folium과 GeoJSON을 활용한 인터랙티브 지도와 Plotly 기반 시각화로 복잡한 데이터를 사용자 친화적으로 전달하는 중요성을 체감.
